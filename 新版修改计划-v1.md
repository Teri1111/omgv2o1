## Phase 1: Prompt 重构 + LLM 真正生效

### P1-T1: 切换 Chat API + KBQA-o1 纯文本格式

**问题**: 当前使用 `/v1/completions` 裸 prompt，deepseek-r1 格式遵循率 <5%。

**方案**: 
1. `llm_agent.py` 的 `_request_completion()` 改用 `/v1/chat/completions`
2. 拆分 prompt 为 `system` + `user` + `assistant` 三段消息
3. **放弃 function calling JSON 格式**，改用 KBQA-o1 的纯文本 Thought/Action/Observation 格式

**新 Prompt 格式** (对标 KBQA-o1):

```
[System Message]
You are a KBQA agent that constructs S-expressions step by step.
Available actions:
- Extract_entity [ entity_name ] → expression = START('mid')
- Find_relation [ relation_name ] → expression = JOIN('relation', expression)  
- Merge [ expressionX | expression ] → expression = AND(expressionX, expression)
- Order [ ARGMAX/ARGMIN | relation ] → expression = ARG('mode', expression, 'relation')
- Compare [ GT/LT/GE/LE | relation ] → expression = CMP('mode', 'relation', expression)
- Time_constraint [ relation | time_value ] → expression = TC(expression, 'relation', 'time')
- Count [ expression ] → expression = COUNT(expression)
- Finish [ expression ] → expression = STOP(expression)

Rules:
1. First action must be Extract_entity
2. Only use relations from the Available Relations list
3. Reply with exactly one Thought and one Action per turn

[User Message]
Question: {question}

Available forward relations: {forward_rels}
Available reverse relations: {reverse_rels}
T5 candidate path hint: {draft_hint}

{scratchpad}
Thought{N}:
```

**验收**: 
- deepseek-r1:14b 在 50 样本上 Action 解析成功率 >50%
- 不再出现 empty response
- ReAct fallback 降至 <30%

**涉及文件**:
- `reasoning/llm_agent.py` — 改 `_request_completion()`, `choose_next_step_function_call()`, `choose_next_step_react()` 合并为统一的 `choose_next_step()`
- `reasoning/agent.py` — 改 `_decide_action()` 适配新返回格式

### P1-T3: Scratchpad 精简

**问题**: Observation 中塞入完整 KB JSON（含 embedding_text、steps 等），单次 observation 可达数千字符。

**方案**:
1. KB consultation observation 只保留 `rule_title` + `suggested_action`，不包含完整 JSON
2. explore_neighbors observation 只保留 relation 列表，不包含完整 entity MID
3. 设置 scratchpad 总长度上限（如 3000 字符），超出时保留首尾，截断中间

**新 Observation 格式**:
```
Observation3: expression = JOIN('film.actor.film', expression)
  [exec: valid, 5 intermediate answers]

Observation4: [KB guidance: "When actor→film returns CVT, add one more hop via film.performance.film"]
```

对标 KBQA-o1 的简洁 Observation：`expression = JOIN('(R film.actor.film)', expression)`

**涉及文件**:
- `reasoning/agent.py` — `_update_scratchpad()` 方法
- `skills/experience_kb_skill.py` — `search_experience_rules()` 返回值精简

### P1-T4: Action 解析器统一重写

**问题**: 当前有三层解析（function call JSON → ReAct regex → heuristic），逻辑复杂且大部分无效。

**方案**: 统一为**单层 KBQA-o1 风格解析器**：

```python
def parse_action(raw_text: str) -> tuple[str, dict]:
    """Parse KBQA-o1 style action from LLM output.
    
    Expected format: 
        Thought: ...
        Action: Find_relation [ film.actor.film ]
    
    Returns: (action_type, params)
    """
    # 1. Strip <think> blocks
    text = re.sub(r'<think>.*?</think>', '', raw_text, flags=re.DOTALL).strip()
    
    # 2. Extract Action line
    m = re.search(
        r'Action\d*\s*:\s*'
        r'(Extract_entity|Find_relation|Merge|Order|Compare|'
        r'Time_constraint|Count|Finish)\s*\[\s*(.+?)\s*\]',
        text, re.IGNORECASE
    )
    if m:
        action_type = m.group(1)
        params_str = m.group(2)
        return _parse_action_params(action_type, params_str)
    
    # 3. Fuzzy: scan for any relation name in text
    return _fuzzy_relation_match(text, available_relations)
```

**涉及文件**: `reasoning/llm_agent.py` — 新 `parse_action()` 替代 `_extract_json_object()` + ReAct regex

## Phase 2: 评估体系补齐

### P2-T1: F1 / Precision / Recall 指标实现

**方案**: 在 `test_closed_loop.py` 和 `data_gen/evaluate_v4.py` 中新增标准 KBQA 评估指标。

```python
def compute_f1(predicted_answers: set, golden_answers: set) -> dict:
    """Standard KBQA F1 computation."""
    if not predicted_answers and not golden_answers:
        return {"precision": 1.0, "recall": 1.0, "f1": 1.0}
    if not predicted_answers or not golden_answers:
        return {"precision": 0.0, "recall": 0.0, "f1": 0.0}
    
    tp = len(predicted_answers & golden_answers)
    precision = tp / len(predicted_answers)
    recall = tp / len(golden_answers)
    f1 = 2 * precision * recall / (precision + recall) if (precision + recall) > 0 else 0.0
    return {"precision": precision, "recall": recall, "f1": f1}
```

**输出格式** (对标论文表格):
```
| Method       | Dataset | F1    | HIT  | P     | R     | EM    |
|-------------|---------|-------|------|-------|-------|-------|
| PG-only     | CWQ     | 45.2  | 58.0 | 42.1  | 52.3  | 35.0  |
| LLM-First   | CWQ     | 52.3  | 67.4 | 48.7  | 59.1  | 40.2  |
| + Best-of-N | CWQ     | 56.8  | 72.1 | 53.4  | 62.8  | 44.5  |
```

**涉及文件**: 
- `test_closed_loop.py` — 新增 `--eval-mode full` 输出 F1

## Phase 3: 搜索机制升级

### P3-T1: Best-of-N 采样

**方案**: 对标 KnowCoder-A1 的 `n_repeat=8`。

```python
def best_of_n(question, n=5, temperature=0.7):
    """Sample N trajectories, return the one with highest reward."""
    trajectories = []
    for i in range(n):
        trace = agent.run(question, temperature=temperature)
        reward = compute_trajectory_reward(trace)
        trajectories.append((trace, reward))
    
    # Sort by reward, return best
    trajectories.sort(key=lambda x: x[1], reverse=True)
    return trajectories[0]

def compute_trajectory_reward(trace):
    """Rule-based reward for trajectory quality."""
    reward = 0.0
    
    # 1. Execution validity
    if trace.final_execution and trace.final_execution.get("valid"):
        reward += 1.0
    
    # 2. Answer count penalty (prefer fewer, more precise answers)
    n_answers = trace.final_execution.get("num_answers", 0)
    if 1 <= n_answers <= 10:
        reward += 0.5
    elif n_answers > 100:
        reward -= 0.5
    
    # 3. Step efficiency (fewer steps = better)
    n_steps = len(trace.steps)
    reward += max(0, (10 - n_steps) * 0.05)
    
    # 4. No hallucinated relations (all used relations in subgraph)
    if not trace.has_hallucinated_relations:
        reward += 0.3
    
    return reward
```

**涉及文件**: 
- `reasoning/agent.py` — 新增 `best_of_n()` 方法
- `reasoning/llm_agent.py` — 支持 `temperature` 参数

### P3-T2: Self-Consistency 投票

**方案**: N 条轨迹各产出一个 S-expression，多数投票选最终答案。

```python
def self_consistency(question, n=5, temperature=0.7):
    """Sample N trajectories, majority vote on final answer."""
    expressions = []
    for i in range(n):
        trace = agent.run(question, temperature=temperature)
        expressions.append(trace.final_sexpr)
    
    # Majority vote (by exact S-expression match)
    counter = Counter(expressions)
    return counter.most_common(1)[0][0]
```

### P3-T3: Trajectory Reward Scoring（轻量级 Reward Model）

**方案**: 不训练独立 reward model，而是用**规则 + LLM 打分**的混合方式。

1. **规则 reward**: execution validity + answer count + step count + relation hallucination check
2. **LLM reward** (可选): 将完整轨迹发给 LLM 打分（1-5 分）

对标 KBQA-o1 的 reward model，但更轻量。

### P4-T2: Cold-Start SFT

**方案**: 对标 KBQA-o1 的 SFT 训练。

1. 基座模型: Qwen2.5-7B-Instruct 或 Llama-3.1-8B-Instruct
2. 训练框架: LlamaFactory（已在 omgv1 中使用）
3. 训练格式: 每条轨迹的每一步切为 (prefix → suffix) SFT 对
4. 训练规模: 5k-10k 轨迹 × 5 步/轨迹 ≈ 25k-50k SFT 样本
5. 预计 4×A100 训练 2-4 小时

**评估**: SFT 后模型在 CWQ test 上的 F1 vs 通用模型的 F1

## 关键依赖与风险

### 依赖

1. **GPU 资源**: Best-of-N（P3-T1）需要 5x 推理时间；SFT（P4-T2）需要 4×A100
2. **Freebase Virtuoso**: CWQ 全集评估需要稳定的 SPARQL 服务
3. **LLM 端点**: 当前 ollama 8002 隧道需要稳定运行

### 风险

1. **P1 失败风险**: 即使改用 Chat API + 纯文本格式，14B 通用模型可能仍无法稳定遵循 KBQA 动作格式。**缓解**: 如果 R1-14B 不行，切换到 Qwen2.5-Coder-14B（历史测试 68% HIT）或更大模型
2. **T5 子图瓶颈**: T5 子图不完整是 MISS 的主因之一，但改善 T5 不在本计划范围内。**缓解**: 在论文中明确标注 T5 oracle 上界
3. **F1 低于预期**: 从 HIT 67.4% 转换为 F1 后，分数可能降至 40-50%。**缓解**: 这是真实指标，但通过 Best-of-N 和 SFT 应该能回升
