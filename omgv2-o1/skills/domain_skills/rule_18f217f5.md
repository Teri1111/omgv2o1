# CVT Intermediate Node Navigation

**Rule Type:** CVT_NAVIGATION  
**Confidence:** 0.8  
**Rule ID:** rule_18f217f5

## When to Apply

Relationship involves Compound Value Types (CVTs) - intermediate nodes that connect entities through mediated relationships (e.g., education records linking person to major field of study).

## Steps

1. Use SearchGraphPatterns to discover the CVT path (e.g., people.person.education -> education.education.major_field_of_study)
2. Write SPARQL with a variable for the intermediate CVT node
3. Chain: ?person ns:people.person.education ?edu_cvt . ?edu_cvt ns:education.education.major_field_of_study ?field
4. Join with type constraints if needed

## Common Cases

- When relationship involves compound value types (cvts) - intermediate nodes that connect entities through mediated relationships (e.g., education records linking person to major field of study).
- Similar scenarios with the same pattern

## Avoid

Trying to directly connect entities that are mediated through CVT nodes.
