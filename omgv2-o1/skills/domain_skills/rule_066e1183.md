# ORDER BY must be outside WHERE block in SPARQL

**Rule Type:** ERROR_RECOVERY  
**Confidence:** 0.95  
**Rule ID:** rule_066e1183

## When to Apply

SPARQL query fails with syntax error at 'ORDER' or 'LIMIT' when placed inside the WHERE clause.

## Steps

1. Close the WHERE block with } before writing ORDER BY
2. Place ORDER BY clause after WHERE: WHERE { ... } ORDER BY ...
3. Place LIMIT clause after ORDER BY: WHERE { ... } ORDER BY ... LIMIT N

## Common Cases

- When sparql query fails with syntax error at 'order' or 'limit' when placed inside the where clause.
- Similar scenarios with the same pattern

## Avoid

Do not put ORDER BY, LIMIT, OFFSET, GROUP BY, or HAVING inside the WHERE { } block.
