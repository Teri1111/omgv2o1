# Explore both forward and reverse paths for two-entity intersection

**Rule Type:** RELATION_DIRECTION  
**Confidence:** 0.9  
**Rule ID:** rule_994fc894

## When to Apply

Question asks what entity X satisfies property of entity A AND property of entity B, where the paths from A and B lead to the same answer.

## Steps

1. Identify the two source entities A and B from the question
2. For each, discover the path to the answer using SearchGraphPatterns
3. Bind both paths to the same ?answer variable
4. Execute SELECT DISTINCT ?answer WHERE { path_A . path_B }

## Common Cases

- When question asks what entity x satisfies property of entity a and property of entity b, where the paths from a and b lead to the same answer.
- Similar scenarios with the same pattern

## Avoid

Querying each path separately and doing application-level intersection, when SPARQL can do it directly.
